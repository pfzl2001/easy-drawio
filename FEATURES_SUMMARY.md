# Easy Draw.io - 扩展核心功能特性总结

本项目是一个深度整合在基于 Chromium 的浏览器中的 Draw.io (diagrams.net) 侧边栏辅助作图插件，特别是为了伴随“AI 模型对话编程/研究”场景而专门量身打造。以下是本项目中您指导我实现的所有核心功能梳理：

## 1. 原生且极简的侧边栏体验 (Side Panel Integration)
* **Manifest V3 架构**：完全符合 Chrome 最新的 V3 扩展标准，安全且高效。
* **合法且轻量引用**：未使用原版极其庞大（50MB+）的源代码，而是合规调用了 JGraph 官方提供的 `embed.diagrams.net` 嵌入式接口。
* **暗黑沉浸式 UI**：为您实现了一套玻璃拟物化 (Glassmorphism) 的顶级质感深色 UI，配有精致的 SVG 矢量图标，完美契合极客审美。

## 2. 独创的“单页隔离”架构 (Tab-Specific Isolation)
* **独立上下文**：与市面上大部分全局侧边栏扩展不同，本插件做到了**侧边栏与当前浏览的标签页 (Tab) 深度绑定**。
* **数据绝不串场**：底层使用 `tabId` 作为哈希键，您在标签 A（比如查阅读书笔记）和标签 B（比如询问 ChatGPT）由于上下文不同，唤出的画板和保存的草稿是完全独立的，不会相互覆盖。
* **自动垃圾回收**：当您关闭特定的浏览器标签页时，`background.js` 后台脚本会自动监听销毁事件，清空该标签页专属的历史无用草稿，避免存储空间膨胀。

## 3. 稳健的本地持久化 & 零隐私追踪 (Local Auto-Sync)
* **自动静默保存**：只要您在画布上有任何拖拽或是修改节点操作，系统会自动通过高频防抖将最新的 XML 序列化保存至 `chrome.storage.local`。
* **刷新防丢失**：即便刷新当前网页或意外短线，依靠这套强绑定的本地存储，重新呼出侧边栏时图形依旧能瞬间恢复。
* **零追踪**：所有数据均仅流转于您的本地浏览器与作图客户端之间，没有任何隐私滥用及云端上传操作。

## 4. 强大的多格式导出系统 (Custom Context Exporting)
* **全格式支持**：您可以选择无损导出为纯正的 `.drawio` (原生格式，双击可用)、透明背景的 `PNG` 图片，或是矢量 `SVG` 数据。
* **智能命名支持**：在点击导出时主动弹出交互对话框供您重命名。并且内置了自动扩展名处理（填不填后缀都不会导致双后缀名错误，如避免 `file.drawio.drawio`），如果留空且取消则自动安全终止操作。

## 5. 多语言“代码转图表”模块 (Code-to-Diagram Module)
* 侧边栏顶部内置了导入入口 `</>`，可以将文本描述的逻辑关系瞬间变成可视化图形，无缝衔接 AI 生态。
* 完美支持直接解析并合并以下三种语言成分：
  1. **Draw.io 原生 XML**
  2. **Mermaid 语言**
  3. **PlantUML 语言**

## 6. 🔥 革命级：AI 回复无感感知与双向联动 (AI XML Auto-Detector)
* **纯事件驱动**：摒弃了极度消耗 CPU 性能的轮询扫描，在 `content.js` 中使用 `MutationObserver` 从底层订阅网页 DOM 树的新增操作。
* **流式输出防抖 (Debounce & Hash Deduplication)**：由于 ChatGPT/Claude 等 AI 输出是字粒度的流式生成，监听器内置了完美的 **1.5 秒渲染静默防抖** 策略，并利用 djb2 字符串哈希算法判断重复。只有当 AI 完全把代码块写完，它才会进行捕获，从不在中间做毫无意义的打断。
* **双模式 UI 反馈**：
  * **Auto-Update ON (开启自动化)**：这是默认模式，AI 写完架构代码的一瞬间，侧边栏画板自动、无感地刷新并直接加载出最新图形，伴随微小友好的绿色 Toast 提示“已从 AI 响应中更新！”。
  * **Auto-Update OFF (手动干预)**：如果您在此期间自己也在作图（不希望被覆盖），关闭顶部自动更新 Toggle 开关后，每当 AI 提供新原文件，则会在画布上方弹出一个不干扰您的“蓝底柔和通知栏”，供您自己决定是否 Click 覆盖，或直接丢弃 Dismiss。
